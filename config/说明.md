将压缩包解压之后找到解压的路径
点击进`交付`文件夹
文件结构如下
```text
├─config
│ └─ mes-config.example.json
│ └─ mes-config.json
├─ skills
│ └─ mes-sop-abnormal-workflows
│   └─SKILL.md
├─ andon_sop-mcp.mjs
├─ 说明.md
```

咱们主要修改`mes-config.json`文件
`mes-config.json`文件内容大致如下:
```json
{
  "mes": {
    "baseUrl": "http://localhost:9998",
    "auth": {
      "type": "login",
      "username": "superadmin",
      "password": "1234554321",
      "loginUrl": "/main/login",
      "contentType": "application/x-www-form-urlencoded"
    },
    "timeout": 300000,
    "queryTimeoutMs": 25000,
    "retry": 3,
    "writeIntervalMs": 2400
  },
  "tools": {
    "likeSelectMaterielList": true,
    "selectCategoryTreeList": true,
    "saveCategory": true,
    "uploadSop": true,
    "queryAbnormalReportPage": true,
    "selectReportFrom": true
  }
}
```

用编辑器打开之后主要修改`baseUrl`字段，`username`字段，`password`字段。
`baseUrl`要修改为远程服务器地址: 对于这个项目来说替换为`http://123.60.102.125:9111/industry`
`username`修改能够登录到mes系统的用户账号
`password`修改为能够能等到mes系统且和username配套的密码
其他的地方不用修改。

等到上面所有的步骤完成之后，再来配置work buddy

```json
{
  "mcpServers": {
    "andon-sop-mcp": {
      "type": "stdio",
      "command": "node",
      "args": [
        "解压路径\\交付\\andon_sop-mcp.mjs"
      ],
      "env": {
        "MES_CONFIG_PATH": "解压路径\\交付\\config\\mes-config.json"
      }
    }
  }
}
```

如果有多个`mcp server`就按照下面这个相对位置粘贴。

```json
{
  "mcpServers": {
    "andon-sop-mcp": {
      "type": "stdio",
      "command": "node",
      "args": [
        "解压路径\\交付\\andon_sop-mcp.mjs"
      ],
      "env": {
        "MES_CONFIG_PATH": "解压路径\\交付\\config\\mes-config.json"
      }
    },

    "another-mcp-server": {
      "type": "stdio",
      "command": "node",
      "args": [
        "D:\\mcp-servers\\another-server\\index.mjs"
      ],
      "env": {
        "CONFIG_PATH": "D:\\mcp-servers\\another-server\\config.json"
      }
    },

    "filesystem-mcp": {
      "type": "stdio",
      "command": "node",
      "args": [
        "D:\\mcp-servers\\filesystem\\server.mjs"
      ]
    }
  }
}
```
