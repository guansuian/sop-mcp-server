1. 先生成 bundle 执行文件

在项目根目录执行：

npm install
npm run bundle

执行成功后，应生成：

安装路径\andon_sop-mcp.mjs

注意：分发给别人使用时，建议使用 .mjs 文件，不要使用 .js 文件。

2. Work Buddy MCP 配置

把下面 JSON 填到 Work Buddy 的自定义 MCP 配置中：

{
  "mcpServers": {
    "andon-sop-mcp": {
      "type": "stdio",
      "command": "node",
      "args": [
        "安装路径\\andon_sop-mcp.mjs"
      ],
      "env": {
        "MES_CONFIG_PATH": "安装路径\\config\\mes-config.json"
      }
    }
  }
}

如果项目目录不同，把 andon_sop-mcp.mjs 和 mes-config.json 的路径改成自己的真实路径。
Windows 路径写到 JSON 中时，\ 要写成 \\。

3. 分发给别人时需要包含这些文件

andon_sop-mcp.mjs
config\mes-config.json
skills\mes-sop-abnormal-workflows\SKILL.md

推荐目录结构：

安装路径
├─ andon_sop-mcp.mjs
├─ config
│  └─ mes-config.json
└─ skills
   └─ mes-sop-abnormal-workflows
      └─ SKILL.md
