# 异常报表流程

## 查询异常报表

用户要查询异常报表记录时，调用 `queryAbnormalReportPage`。

用户要查询异常解决方案时，调用 `queryAbnormalReportPage`。

用户要查询异常情况时，调用 `queryAbnormalReportPage`。

该工具内部会处理下面流程：

1. 查询动态表单字段配置。
2. 根据可搜索字段构建查询参数。
3. 分页查询异常报表真实数据。
4. 查询 select 字段需要的数据字典。
5. 使用字典 label 格式化展示数据。

## 构建查询条件

以工具返回的 searchable fields 为准。

规则：

- 过滤条件 key 使用字段的 `field`，不要使用中文标题。
- `varchar` 等文本字段支持模糊查询。
- 日期时间字段支持范围查询，格式为 `{ "start": "...", "end": "..." }`。
- select 字段筛选时使用 `dictMap` 中的字典 value，不要用 label。
- 不确定字段时，先查询字段配置，不要猜字段名。

## 查询结果使用方式

Markdown 表格用于快速展示给用户。

Work Buddy 查询上下文用于后续操作，里面包含：

- 展示记录。
- 原始记录 id。
- 可搜索字段。
- 字典映射。

不要猜测状态、类型等字段的含义。向用户解释结果时，优先使用工具已经翻译好的展示值。

## HTML 报表生成规则

异常报表不要只停留在 Markdown 表格。第一次完成异常报表查询后，应根据本次查询结果生成一个本地 HTML 可视化报表，并给用户一个可点击或可打开的 HTML 文件链接。

生成策略：

- 同一轮异常报表查询中，第一次查询后生成 HTML。
- 用户继续翻页、微调查看同一批查询结果时，默认不要重复生成 HTML。
- 如果用户明确说“重新生成 HTML”“再生成一份报表”“我要 HTML 页面”等强需求，可以再次生成 HTML。
- 如果查询结果为空，也要生成空状态 HTML 页面，而不是只返回“无数据”。

HTML 文件要求：

- 输出完整 HTML，包括 `<!DOCTYPE html>`、`<html>`、`<head>`、`<body>`。
- CSS 写在 `<style>` 标签中。
- JavaScript 写在 `<script>` 标签中。
- 默认不要依赖外部 CDN。
- 不要使用需要联网加载的图片、字体或图表库。
- 图表优先用纯 HTML/CSS 实现，例如横向进度条。
- 页面应能在本地浏览器直接打开。
- 最终给用户的是 HTML 文件链接，不要只输出 Markdown 表格。

## HTML 页面结构

HTML 报表建议按下面顺序组织：

1. 标题区。
2. 总览统计卡片。
3. 异常类型分布。
4. 工位异常分布。
5. 异常时间线。
6. 异常详情卡片列表。
7. 原始数据摘要或备注。

标题区展示：

- 报表标题，例如“安灯异常处理报表”。
- 生成时间。
- 数据总数。
- 数据来源说明。

总览统计卡片至少展示：

- 异常总数。
- 已关闭数量。
- 未关闭数量。
- 平均处理时长。
- 最长处理时长。
- 涉及工位数量。

异常类型分布：

- 按 `abnormalType` 分组统计数量。
- 如果没有图表库，用 CSS 横向进度条展示。

工位异常分布：

- 按 `station` 分组统计数量。

异常时间线：

- 按 `beginOccurDate` 从早到晚排序。
- 每条时间线展示发生时间、工位、异常类型、异常详情摘要、持续时长、关闭人。

异常详情卡片：

- 每条异常记录生成一个卡片。
- 展示异常类型、工位、操作人、开始时间、关闭时间、持续时长、异常描述、处理结果。
- `handleResult` 如果包含多行步骤，按换行拆成 `<ol><li>...</li></ol>`。

## HTML 数据处理规则

生成 HTML 前，先对 `data` 数组做统计：

- `total = data.length`
- 已关闭数量：`closeStatus === "1"`
- 未关闭数量：`closeStatus !== "1"`
- 涉及工位数量：按 `station` 去重。
- 异常类型数量：按 `abnormalType` 分组。

`duration` 可能是类似 `0天0时2分36秒` 的文本。应尽量解析为秒数，用于平均时长、最长时长和排序。如果解析失败，保留原始文本，不要报错。

字段缺失时，HTML 中显示“未提供”，不要编造数据。

## 异常类型展示

如果工具返回的数据字典里已经有 `abnormalType` 的真实含义，优先使用数据字典。

如果没有数据字典，可以使用默认映射：

| abnormalType | 默认含义 | 颜色建议 |
|---|---|---|
| `0` | 设备异常 | 红色或橙红色 |
| `1` | 生产节拍异常 | 橙色或黄色 |
| `2` | 质量/物料异常 | 紫色或蓝紫色 |
| 其他 | 未知异常 | 灰色 |

## 视觉风格

HTML 页面应符合工业 MES 看板风格：

- 清晰、专业，适合管理者查看。
- 使用浅色背景。
- 使用卡片布局。
- 重点数字突出。
- 异常类型使用不同颜色标签。
- 处理步骤使用列表、时间线或步骤条。
- 页面宽度适合 PC 浏览器。

## 给用户的回复方式

完成查询和 HTML 生成后，回复用户时应包含：

- 简短说明查询结果。
- HTML 文件链接。
- 如果仍有分页，说明是否有上一页或下一页。

不要把完整 HTML 源码直接贴到聊天里，除非用户明确要求查看源码。
